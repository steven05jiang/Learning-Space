# Tech Debt — Implementation Shortcuts

Lightweight log of shortcuts, stubs, and deferred improvements noted **during implementation**.
Written by the implementer agent when a known-imperfect choice is made to keep momentum.

For formal tracked TD- tasks (with full PM workflow), see `memory/tech-debt-tracker.md`.

---

## Format

Each entry:

```
### YYYY-MM-DD | <TASK-ID> | <short title>
**Shortcut:** <what was done instead of the right thing>
**Proper fix:** <what should be done later>
**Why deferred:** <reason — time, blocked dependency, out of scope>
**Priority:** P0 (blocking) / P1 (before release) / P2 (nice to have)
```
