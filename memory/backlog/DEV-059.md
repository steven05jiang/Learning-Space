# DEV-059: Add manual Re-process action to resource detail UI

**Status:** ⏳ Pending
**Feedback:** FB-002
**Priority:** HIGH | **Effort:** S
**Depends on:** DEV-057, DEV-043 ✅

## Description

Add a "Re-process" button to the resource detail page that lets users manually trigger reprocessing of a resource. This is the only way to retry a failed resource or force a re-summarization. The button resets `processing_status` to `pending` and enqueues a new worker job.

## Acceptance Criteria

- [ ] `POST /resources/{id}/reprocess` endpoint added; resets `processing_status = pending` and enqueues worker job; returns 202
- [ ] Endpoint returns 401 if unauthenticated, 403 if resource belongs to another user
- [ ] "Re-process" button visible on resource detail page
- [ ] Button is disabled (grayed out) while `processing_status = processing`
- [ ] Button click calls POST /resources/{id}/reprocess; shows toast confirmation on 202
- [ ] Resource detail page refreshes `processing_status` after re-process is triggered
- [ ] Unit test for new endpoint (success, 401, 403)
