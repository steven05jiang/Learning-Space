## Description

Register a domain on Namecheap and configure Cloudflare as the DNS provider. Point the apex domain to the Vercel frontend deployment and an `api.*` subdomain to the Railway backend. Update Google OAuth redirect URIs to use the custom domain.

## Acceptance Criteria

- [ ] Domain registered on Namecheap; Cloudflare added as nameserver
- [ ] Apex domain (`example.com`) CNAME/A record points to Vercel; SSL active
- [ ] `api.example.com` CNAME points to Railway; SSL active
- [ ] Vercel project custom domain configured and verified
- [ ] Railway custom domain configured and verified
- [ ] Google OAuth redirect URIs updated to use `https://example.com/auth/callback/google`
- [ ] `NEXT_PUBLIC_API_URL` updated to `https://api.example.com` in Vercel env vars
- [ ] End-to-end: login at `https://example.com` → Google OAuth → redirect back to app
- [ ] `memory/infra.md` updated with final domain/subdomain records
