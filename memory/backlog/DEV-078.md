## Description

Phase 2 schema foundation. Add an Alembic migration that enables the pgvector extension and creates the `resource_embeddings` table with an IVFFlat approximate nearest-neighbor index. The table stores one embedding vector per resource; the FK `ON DELETE CASCADE` ensures automatic cleanup when a resource is deleted.

## Acceptance Criteria

- [ ] Migration enables `CREATE EXTENSION IF NOT EXISTS vector`
- [ ] `resource_embeddings` table created: `resource_id UUID PK FK(resources.id) ON DELETE CASCADE`, `embedding vector(1536) NOT NULL`, `model VARCHAR(100) NOT NULL`, `created_at / updated_at TIMESTAMPTZ`
- [ ] IVFFlat index created: `USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100)`
- [ ] `alembic upgrade head` and `alembic downgrade` both clean
- [ ] Existing migration chain unbroken
