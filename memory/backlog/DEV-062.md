# DEV-062: Update LLM prompt for tag reuse and top_level_categories

**Status:** ⏳ Pending
**Feedback:** FB-003
**Priority:** HIGH | **Effort:** S
**Depends on:** DEV-022 ✅, DEV-060
**Design spec:** `docs/design-category-taxonomy.md`

## Description

Update the LLM processing prompt to:
1. Include the user's existing tag list as context so the model can reuse tags instead of generating new variants
2. Include the valid root category names (from the `categories` table) so the model can assign them
3. Require a `top_level_categories` field in the LLM JSON output (non-empty list of valid root category names)

Add validation of LLM output against the categories table; raise structured errors on invalid or missing categories.

## Acceptance Criteria

- [ ] LLM prompt includes the user's existing tag list (fetched from Neo4j before LLM call)
- [ ] LLM prompt includes the list of valid root category names (fetched from `categories` table)
- [ ] LLM JSON output schema requires `top_level_categories` (list of strings, at least one entry)
- [ ] Post-LLM validation: if any value in `top_level_categories` is not a valid category name → raise `INVALID_CATEGORY` error, resource marked `failed`
- [ ] Post-LLM validation: if `top_level_categories` is empty or missing → raise `CATEGORY_REQUIRED` error, resource marked `failed`
- [ ] `resources.top_level_categories` JSONB field stored after successful LLM call
- [ ] Unit tests: valid output, INVALID_CATEGORY, CATEGORY_REQUIRED
