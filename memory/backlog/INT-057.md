# INT-057: User filters search results by tag — results narrowed to matching tag

**Status:** ⏳ Pending
**Feedback:** N/A
**Priority:** MEDIUM | **Effort:** S
**Depends on:** DEV-073, DEV-074
**Design spec:** docs/design-search.md §4.2, §6.2

## Description

BDD integration test verifying the `?tag=` filter parameter. Confirms that when a tag filter is provided, only resources containing that exact tag are returned (JSONB `?` operator), even when multiple resources match the keyword query.

## Acceptance Criteria

- [ ] Test creates READY resources: some with target tag, some without
- [ ] `GET /resources/search?q=<keyword>&tag=<tag>` returns only resources containing that exact tag
- [ ] Resources matching keyword but missing the tag are excluded from results
- [ ] `?tag=nonexistent` returns empty list (total=0), not an error
- [ ] Test marked `@pytest.mark.integration` and assigned to `search` CI group
