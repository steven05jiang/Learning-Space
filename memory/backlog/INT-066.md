# INT-066: Quick-add creates resource, sets is_added=true; post cleaned up if fully added

**Status:** ⏳ Pending
**Feedback:** N/A
**Priority:** HIGH | **Effort:** M
**Depends on:** DEV-085, DEV-083
**Design spec:** `docs/design-twitter-integration.md` §6.5, §4 (cleanup)

## Description

Integration test verifying the full quick-add flow: `POST /discover/bookmarks/{tweet_id}/add` creates a resource via the internal `POST /resources` flow, sets `is_added=true` on the bookmark, and returns 202 with the resource. Also verifies idempotency (200 if already added) and 404 on unknown tweet_id. Post cleanup: when all bookmarks for a post are `is_added=true`, the `twitter_posts` row is eligible for deletion.

## Acceptance Criteria

- [ ] `POST /discover/bookmarks/{tweet_id}/add` returns 202 with resource object (status PROCESSING)
- [ ] Resource `original_content` equals the `tweet_url`; `prefer_provider=twitter`
- [ ] `twitter_bookmarks.is_added` set to `true` after successful resource creation
- [ ] Calling again when `is_added=true` returns 200 (idempotent no-op, no duplicate resource)
- [ ] Returns 404 if `tweet_id` not in user's bookmarks
- [ ] After all users' bookmarks for a post are `is_added=true`, cleanup query removes the `twitter_posts` row
