## Description

Add pytest unit tests (no real DB) for `ResourceSearchService` and the `GET /resources/search` endpoint. Uses mocked `AsyncSession` and mocked service responses. Covers the critical paths: successful search, tag filtering, READY-only enforcement, pagination, and validation error codes.

## Acceptance Criteria

- [ ] `pytest -m "not integration"` passes with new tests included
- [ ] Test: keyword match returns ranked results in `ts_rank` DESC order
- [ ] Test: `tag=` filter param is passed to service; results narrowed
- [ ] Test: non-READY resources not returned (READY-only filter)
- [ ] Test: `q=""` → 400 `SEARCH_QUERY_EMPTY`
- [ ] Test: `q` > 500 chars → 400 `SEARCH_QUERY_TOO_LONG`
- [ ] Test: unauthenticated request → 401
- [ ] Test: `rank` field present in each result
- [ ] Test: `total` reflects full match count, not page size
