**Status:** ⏳ Pending

## Description

Phase 2 search upgrade. Extend `ResourceSearchService` with `_vector_search()` (pgvector cosine distance) and `_hybrid_search()` (runs full-text + vector in parallel, merges via RRF with k=60). A `SEARCH_MODE` env var selects the active strategy: `"fulltext"` (Phase 1, default) or `"hybrid"` (Phase 2). HTTP endpoint and agent tool contracts are unchanged.

## Acceptance Criteria

- [ ] `_vector_search(session, owner_id, query_embedding, tag, limit)` implemented using `<=>` cosine distance operator
- [ ] `_hybrid_search()` runs `_full_text_search` and `_vector_search` in parallel (asyncio.gather)
- [ ] RRF merge implemented with k=60: `score(d) = Σ 1/(k + rank_i(d))`
- [ ] Results sorted by combined RRF score DESC
- [ ] `SEARCH_MODE` env var: `"fulltext"` → Phase 1 path, `"hybrid"` → Phase 2 path
- [ ] HTTP endpoint and agent tool contracts unchanged (no schema changes)
- [ ] Phase 1 behavior unaffected when `SEARCH_MODE=fulltext`
- [ ] Unit tests for `_hybrid_search` using mocked retrieval results
