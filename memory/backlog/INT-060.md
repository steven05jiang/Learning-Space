# INT-060: Twitter integration status reflects all connection variants

**Status:** ⏳ Pending
**Feedback:** N/A
**Priority:** HIGH | **Effort:** S
**Depends on:** DEV-082
**Design spec:** `docs/design-twitter-integration.md` §6.1

## Description

Integration test verifying that `GET /integrations/twitter/status` returns the correct response for each of the three connection states: not connected, connected without `bookmark.read` scope, and fully connected.

## Acceptance Criteria

- [ ] With no Twitter account linked: `connected=false`, all other fields null/0
- [ ] With Twitter linked but `token_scopes` missing `bookmark.read`: `connected=true`, `has_bookmark_scope=false`
- [ ] With Twitter linked and `token_scopes` containing `bookmark.read`: `connected=true`, `has_bookmark_scope=true`
- [ ] `pending_bookmark_count` reflects actual count of `is_added=false` bookmarks for user
- [ ] Returns 401 if unauthenticated
