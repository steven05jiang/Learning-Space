# DEV-057: Add processing_status field to resources + Alembic migration

**Status:** ⏳ Pending
**Feedback:** FB-002
**Priority:** HIGH | **Effort:** S
**Depends on:** DEV-002 ✅

## Description

Add a `processing_status` enum field to the `resources` table to explicitly track whether a resource has been processed. This replaces implicit status inference and enables the worker to skip already-processed resources and users to manually trigger reprocessing.

Valid states: `pending` → `processing` → `success` | `failed`

## Acceptance Criteria

- [ ] `processing_status` enum column added to `resources` table (values: `pending`, `processing`, `success`, `failed`; default: `pending`)
- [ ] Alembic migration created; applies cleanly on fresh and existing databases
- [ ] Existing rows backfilled: status=`READY` → `success`, status=`FAILED` → `failed`, all others → `pending`
- [ ] Pydantic `Resource` response schema exposes `processing_status`
- [ ] Unit test: migration idempotency and backfill correctness
