## Description

Create the PostgreSQL `categories` table for root-level topic groupings. Seed 10 system categories on migration. Expose GET/POST/DELETE /categories endpoints so the frontend and LLM pipeline can read and manage categories.

Seeded system categories (is_system=true): Technology, Science, Business, Arts, Health, Education, Politics, Entertainment, Sports, Philosophy

## Acceptance Criteria

- [ ] `categories` table created via Alembic migration: columns `id`, `name`, `is_system` (bool), `owner_id` (nullable FK to users; null for system categories), `created_at`
- [ ] 10 system categories seeded in migration (idempotent — safe to re-run)
- [ ] `GET /categories` returns all categories visible to the user (all system + own user-created); no auth required to list system categories
- [ ] `POST /categories` creates a user-owned category; name must be unique per user; returns 409 on duplicate
- [ ] `DELETE /categories/{id}` deletes a user-owned category; returns 403 if attempting to delete a system category; returns 404 if not found or not owned by user
- [ ] Unit tests for all three endpoints (success paths + error cases)
