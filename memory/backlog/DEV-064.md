# DEV-064: Tag editor component in resource detail UI

**Status:** ⏳ Pending
**Feedback:** FB-004
**Priority:** MEDIUM | **Effort:** S
**Depends on:** DEV-043 ✅, DEV-061

## Description

Add a tag editor to the resource detail page so users can add and remove tags after a resource is processed. Saving the updated tag set calls PATCH /resources/{id} with the new tags array and enqueues a graph resync job for that resource to keep edges consistent.

## Acceptance Criteria

- [ ] Resource detail page displays current tags as chips with a × remove button on each
- [ ] Tag input field allows adding new tags (confirmed with Enter or comma)
- [ ] "Save tags" button calls `PATCH /resources/{id}` with the updated `tags` array
- [ ] Backend enqueues a graph resync job on tag change (update Neo4j edges to reflect new tag set)
- [ ] At least one root-level category tag must remain after editing; UI shows validation error if the last root category tag is removed
- [ ] Toast confirmation shown on successful save
- [ ] Save button disabled when no changes have been made
