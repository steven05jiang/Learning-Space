## Description

Add an email allowlist gate to the OAuth callback flow. After a successful Google OAuth authentication, check the user's verified email against a comma-separated ALLOWED_EMAILS environment variable. If the email is not in the list, do not create a session — instead return a redirect to `/coming-soon`. This enables the owner to manually control who can access the app by updating a single env var.

## Acceptance Criteria

- [ ] `ALLOWED_EMAILS` env var read at startup (comma-separated gmail addresses, e.g. `a@gmail.com,b@gmail.com`)
- [ ] OAuth callback checks user's verified email against allowlist after successful provider auth
- [ ] If email not in allowlist: session is NOT created; response redirects to `/coming-soon`
- [ ] If email is in allowlist: flow proceeds normally (session created, redirect to app)
- [ ] If `ALLOWED_EMAILS` is empty/unset: all authenticated users are allowed (fail-open for dev)
- [ ] X/Twitter OAuth callback also subject to the same check (even if login UI button is removed)
- [ ] Unit test: allowlisted email → allowed; non-allowlisted email → redirect; empty env → allowed
- [ ] `make api-lint` passes
