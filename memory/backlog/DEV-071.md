## Description

Refactor the LLM service layer into a provider-agnostic interface that supports Anthropic Claude (existing default), Groq, SiliconFlow, and Fireworks AI. The latter three are all OpenAI-compatible APIs (same SDK, different base_url + api_key). The active provider is selected via an `LLM_PROVIDER` env var. The LangGraph agent (DEV-032) must use the same abstraction so all LLM calls are routed through a single switchable client.

## Acceptance Criteria

- [ ] `LLM_PROVIDER` env var accepted: `anthropic` (default), `groq`, `siliconflow`, `fireworks`
- [ ] `LLM_MODEL` env var accepted to override the default model per provider
- [ ] Provider-specific base URLs and default models documented in `.env.example`
  - Groq: `https://api.groq.com/openai/v1`
  - SiliconFlow: `https://api.siliconflow.cn/v1`
  - Fireworks: `https://api.fireworks.ai/inference/v1`
- [ ] LLM processor (title/summary/tag generation) uses abstraction — no direct Anthropic SDK import in business logic
- [ ] LangGraph agent uses same abstraction for chat LLM calls
- [ ] Anthropic provider continues to work with existing `ANTHROPIC_API_KEY`; OpenAI-compatible providers use `LLM_API_KEY`
- [ ] Unit tests pass with mocked provider (provider switching does not require live credentials in CI)
- [ ] `make api-lint` passes; `make api-test` passes
