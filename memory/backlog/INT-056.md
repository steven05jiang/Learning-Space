# INT-056: User searches for resources by keyword — returns ranked READY resources only

**Status:** ⏳ Pending
**Feedback:** N/A
**Priority:** MEDIUM | **Effort:** S
**Depends on:** DEV-073, DEV-074
**Design spec:** docs/design-search.md §8.1, docs/integration-test-design.md

## Description

BDD integration test for the core keyword search flow. Verifies that `GET /resources/search?q=<keyword>` returns a ranked list of the user's READY resources, excludes non-READY resources, and includes the `rank` field. Uses a real PostgreSQL test database with the GIN index in place.

## Acceptance Criteria

- [ ] Test creates a set of READY resources with different titles/summaries/tags
- [ ] Test creates at least one PENDING or FAILED resource (must be excluded)
- [ ] `GET /resources/search?q=<keyword>` returns 200 + `{resources: [...], total: N}`
- [ ] Returned resources are READY only
- [ ] Results are ordered by relevance (rank DESC)
- [ ] `rank` field present and > 0 on each result
- [ ] Resources belonging to another user are not returned
- [ ] Test marked `@pytest.mark.integration` and assigned to `search` CI group
