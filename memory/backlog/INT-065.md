# INT-065: Discover list returns unprocessed bookmarks sorted by bookmarked_at DESC

**Status:** ⏳ Pending

**Design spec:** `docs/design-twitter-integration.md` §6.4

## Description

Integration test verifying that `GET /discover/bookmarks` returns only `is_added=false` bookmarks for the authenticated user, joined with `twitter_posts`, sorted by `bookmarked_at DESC`, with correct pagination.

## Acceptance Criteria

- [ ] Returns only `is_added=false` bookmarks for the current user
- [ ] Items sorted by `bookmarked_at DESC` (newest first)
- [ ] Items include `tweet_id`, `tweet_url`, `author_name`, `author_username`, `title`, `preview_text`, `posted_at`, `bookmarked_at`
- [ ] `total` reflects count of all `is_added=false` rows; `limit`/`offset` respected
- [ ] `is_added=true` bookmarks excluded from results
- [ ] Other users' bookmarks not included
- [ ] Returns 401 if unauthenticated
