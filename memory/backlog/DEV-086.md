## Description

Add an "Integrations" sub-section to the Settings page. The panel renders three states driven by `GET /integrations/twitter/status`: (A) not connected — shows "Connect X.com" button; (B) connected without `bookmark.read` — shows ⚠ warning and "Grant Bookmark Access" button; (C) fully connected — shows ✓ confirmation, last-synced time, and pending count. Connect and re-auth flows call `POST /integrations/twitter/authorize` and redirect the user to the returned `redirect_url`. Disconnect shows a confirmation dialog before calling `DELETE /integrations/twitter/disconnect`.

## Acceptance Criteria

- [ ] Settings page has an "Integrations" sub-section
- [ ] Fetches `GET /integrations/twitter/status` on mount; shows loading skeleton during fetch
- [ ] **State A** (`connected=false`): displays "Connect X.com" button; clicking initiates OAuth redirect
- [ ] **State B** (`connected=true, has_bookmark_scope=false`): displays `@username`, ⚠ warning text, "Grant Bookmark Access" button
- [ ] **State C** (`connected=true, has_bookmark_scope=true`): displays `@username`, ✓ confirmation, last-synced relative time, pending bookmark count, "Bookmarks sync automatically every hour" note
- [ ] "Disconnect" button visible in States B and C
- [ ] Disconnect shows confirmation dialog: "This will remove bookmark sync and clear all your Discover items. Your added resources will not be affected."
- [ ] After confirmed disconnect: `DELETE /integrations/twitter/disconnect` called; UI transitions to State A
- [ ] OAuth redirect: `POST /integrations/twitter/authorize` called; `redirect_url` used for `window.location.href`
- [ ] After OAuth callback returns to Settings: panel re-fetches status and updates state
