## Description

Add an Alembic migration that creates a functional GIN index on the `resources` table for Phase 1 full-text search. The index covers `title`, `summary`, and `tags` (cast to text) via a `to_tsvector` expression. No new column is required — this is a pure index migration. `CREATE INDEX CONCURRENTLY` avoids table lock in production.

## Acceptance Criteria

- [ ] Alembic migration file created: `add_resources_search_idx`
- [ ] `alembic upgrade head` runs clean
- [ ] Index `resources_search_idx` exists on the `resources` table post-migration using `GIN (to_tsvector('english', COALESCE(title,'') || ' ' || COALESCE(summary,'') || ' ' || COALESCE(tags::text,'[]')))`
- [ ] `alembic downgrade` removes the index
- [ ] Existing migration chain is unbroken
