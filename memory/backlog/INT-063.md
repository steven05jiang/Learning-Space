## Description

Integration test verifying the core bookmark sync flow: given a user with `bookmark.read` scope and a mocked Twitter Bookmarks API response, the cron task creates `twitter_posts` rows for new posts and `twitter_bookmarks` rows for the user.

## Acceptance Criteria

- [ ] Running sync for a user with `bookmark.read` scope and mocked API response creates `twitter_posts` rows for each new post
- [ ] `twitter_bookmarks` rows created with correct `user_id`, `tweet_id`, `bookmarked_at`
- [ ] Posts older than 7 days are not fetched or stored
- [ ] Users without `bookmark.read` scope are skipped entirely
