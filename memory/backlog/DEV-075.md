# DEV-075: Add search_resources tool to LangGraph agent + update system prompt

**Status:** ⏳ Pending
**Feedback:** N/A
**Priority:** HIGH | **Effort:** S
**Depends on:** DEV-032 ✅, DEV-073
**Design spec:** docs/design-search.md §7

## Description

Add the `search_resources` LangGraph tool to the existing agent. The tool wraps `ResourceSearchService.search()` with a hard `limit=10` cap and `offset=0` (no pagination in agent context). Returns `List[AgentResourceResult]` — the trimmed shape that strips status, timestamps, and fetch metadata. Also updates the agent system prompt with the broader-query guidance from the design spec.

## Acceptance Criteria

- [ ] `@tool async def search_resources(query: str, tag: Optional[str] = None)` added to agent tool list
- [ ] Tool calls `ResourceSearchService.search(owner_id, query, tag, limit=10, offset=0)`
- [ ] Returns `List[dict]` serialized from `AgentResourceResult` (id, title, summary, tags, top_level_categories, url)
- [ ] `url` is `original_content` when `content_type == "url"`, else `None`
- [ ] Hard limit=10 enforced regardless of caller
- [ ] Agent system prompt updated with: "When searching for resources, prefer broader queries over narrow ones... If search_resources returns an empty list, try a single-keyword version of the query before reporting no results."
- [ ] `current_user.id` (owner_id) correctly scoped from agent invocation context
- [ ] Existing agent unit tests still pass
