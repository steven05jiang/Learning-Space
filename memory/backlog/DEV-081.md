**Status:** ⏳ Pending

## Description

Single Alembic migration adding: (1) `token_scopes TEXT` column (nullable) to `user_accounts`, (2) `twitter_posts` global post cache table (PK: `tweet_id`), (3) `twitter_bookmarks` per-user bookmark table with `(user_id, tweet_id)` unique constraint. All indexes per design spec.

## Acceptance Criteria

- [ ] Alembic migration applies cleanly (`upgrade` and `downgrade`)
- [ ] `user_accounts.token_scopes` column exists, nullable TEXT
- [ ] `twitter_posts` table exists with all columns: `tweet_id` (PK), `tweet_url`, `author_name`, `author_username`, `title`, `preview_text`, `posted_at`, `fetched_at`, `created_at`
- [ ] `twitter_bookmarks` table exists with all columns: `id` (UUID PK), `user_id` (FK users.id), `tweet_id` (FK twitter_posts.tweet_id), `bookmarked_at`, `first_fetched_at`, `is_added` (default false), `created_at`
- [ ] Unique constraint `(user_id, tweet_id)` on `twitter_bookmarks` enforced
- [ ] Index on `(user_id, is_added, bookmarked_at DESC)` on `twitter_bookmarks`
- [ ] Index on `first_fetched_at` on `twitter_bookmarks`
- [ ] Index on `tweet_id` on `twitter_bookmarks`
