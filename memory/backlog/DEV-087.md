# DEV-087: Discover page UI

**Status:** ⏳ Pending
**Feedback:** N/A
**Priority:** HIGH | **Effort:** M
**Depends on:** DEV-085, DEV-086
**Design spec:** `docs/ux-requirements.md` §7 (sidebar badge), §7.2 (Discover page)

## Description

Add the "Discover" sidebar nav item with a numeric badge (shown when `pending_bookmark_count > 0`; hidden entirely when `has_bookmark_scope=false`). Implement the Discover page: paginated bookmark card list with [View] and [+ Add] buttons, [+ Add] button loading/success/error states, empty state, skeleton loading state, and a no-scope prompt state for users without bookmark permission.

## Acceptance Criteria

- [ ] Sidebar "Discover" item hidden when `has_bookmark_scope=false`
- [ ] Sidebar "Discover" item visible with numeric badge when `pending_bookmark_count > 0`
- [ ] Page fetches `GET /discover/bookmarks` with `limit=20&offset=0` on load
- [ ] Bookmark cards display: title (Article title or `@username` for tweets), `@username · date`, `preview_text` (with `...` if truncated at 200 words), [View] and [+ Add] buttons
- [ ] [View] opens `tweet_url` in a new tab
- [ ] [+ Add] **default**: "+ Add" label
- [ ] [+ Add] **loading**: spinner, button disabled (prevents double-submit)
- [ ] [+ Add] **success**: "Added" label, muted, non-interactive; card fades out and is removed after 1.5 s
- [ ] [+ Add] **error**: button re-enabled; toast notification with error message from API
- [ ] Pagination controls (Prev / Page N of M / Next) functional; updates on page change
- [ ] **Empty state**: icon + "Your bookmarks are all caught up. New bookmarks sync every hour."
- [ ] **Loading state**: 3 skeleton cards during initial fetch
- [ ] **No-scope state** (direct URL, no bookmark permission): "Connect your X.com account to start discovering bookmarks. [Go to Settings →]"
