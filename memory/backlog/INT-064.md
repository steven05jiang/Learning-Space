# INT-064: Sync skips already-cached posts — no duplicate twitter_posts rows

**Status:** ⏳ Pending
**Feedback:** N/A
**Priority:** MEDIUM | **Effort:** S
**Depends on:** DEV-084
**Design spec:** `docs/design-twitter-integration.md` §5.2

## Description

Integration test verifying that when a `tweet_id` already exists in `twitter_posts`, the sync does not call the Twitter content API again — only the `twitter_bookmarks` insert is made (idempotent via ON CONFLICT DO NOTHING).

## Acceptance Criteria

- [ ] Pre-existing `twitter_posts` row for a `tweet_id` is not overwritten or duplicated
- [ ] `twitter_bookmarks` row is still created for the user if not already present
- [ ] No additional Twitter API calls are made for already-cached posts (verifiable via mock call count)
- [ ] Running sync twice for the same user produces no duplicate `twitter_bookmarks` rows
