## Description

BDD integration test verifying the `search_resources` LangGraph tool returns correctly shaped `AgentResourceResult` objects. Confirms the trimmed shape (no status/timestamps/rank), the hard limit=10 cap, and correct `url` field behavior for URL vs text resources.

## Acceptance Criteria

- [ ] Tool invocation returns list of dicts with fields: `id`, `title`, `summary`, `tags`, `top_level_categories`, `url`
- [ ] No `rank`, `status`, `created_at`, `updated_at` fields in result
- [ ] `url` is the `original_content` value for URL-type resources
- [ ] `url` is `None` for text-type resources
- [ ] Hard limit: at most 10 results returned even when more match
- [ ] Results scoped to the current user's resources only
- [ ] Test marked `@pytest.mark.integration` and assigned to `search` CI group
