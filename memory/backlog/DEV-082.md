## Description

Implement `GET /integrations/twitter/status`, `POST /integrations/twitter/authorize`, and `DELETE /integrations/twitter/disconnect`. Add `has_scope(account, scope) → bool` utility. The authorize endpoint builds the Twitter OAuth 2.0 URL with the full 4-scope set (`tweet.read users.read bookmark.read offline.access`). The OAuth callback at `/auth/callback` is extended to handle `state=twitter_bookmark_auth:<user_id>` and update `user_accounts.token_scopes` on success.

## Acceptance Criteria

- [ ] `has_scope(account, scope) → bool` correctly parses space-separated `token_scopes`; returns false if `token_scopes` is NULL
- [ ] `GET /integrations/twitter/status` returns all fields: `connected`, `has_bookmark_scope`, `granted_scopes`, `account` (`display_name`, `username`), `last_synced_at`, `pending_bookmark_count`
- [ ] Status correctly reflects three states: not connected / connected without `bookmark.read` / fully connected
- [ ] `POST /integrations/twitter/authorize` returns `{ redirect_url }` pointing to Twitter OAuth with all 4 scopes
- [ ] `/auth/callback` handles `state=twitter_bookmark_auth:<user_id>`; updates `access_token`, `refresh_token`, `token_scopes` on the existing `user_accounts` row
- [ ] `DELETE /integrations/twitter/disconnect` clears `token_scopes` on the `user_accounts` row; deletes all `twitter_bookmarks` for the user; unlinks account entirely if not used for login; returns 204
- [ ] All endpoints require authentication (401 if unauthenticated)
