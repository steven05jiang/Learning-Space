## Description

BDD integration test for search input validation edge cases. Verifies that the endpoint rejects empty queries and queries exceeding the 500-character limit with the correct error codes.

## Acceptance Criteria

- [ ] `GET /resources/search?q=` → 400 with error code `SEARCH_QUERY_EMPTY`
- [ ] `GET /resources/search?q=<501+ chars>` → 400 with error code `SEARCH_QUERY_TOO_LONG`
- [ ] `GET /resources/search` (no `q` param) → 422 (FastAPI required field validation)
- [ ] Test marked `@pytest.mark.integration` and assigned to `search` CI group
