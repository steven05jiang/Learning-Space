## Requirements

Implement the `ResourceSearchService` class with `_full_text_search()` using PostgreSQL `tsvector @@ plainto_tsquery` and `ts_rank`. This is the single shared backend called by both the HTTP endpoint (DEV-074) and the LangGraph agent tool (DEV-075). Also defines the internal models: `SearchResult`, `ResourceSearchItem`, and `AgentResourceResult`.

### Acceptance Criteria

- [x] `ResourceSearchService` class created with `async def search(owner_id, query, tag, limit, offset) -> SearchResult`
- [x] `_full_text_search()` implements the SQL from `docs/design-search.md §4.3` using `sqlalchemy.text()`
- [x] Only `status = 'READY'` resources returned
- [x] Results ordered by `ts_rank DESC`
- [x] Optional `tag` filter uses JSONB `?` operator (escaped as `??` in SQLAlchemy)
- [x] `COUNT(*) OVER()` used for total count (single query)
- [x] `SearchResult` dataclass: `resources: List[ResourceSearchItem]`, `total: int`
- [x] `ResourceSearchItem` dataclass: all fields from design spec §2.3 including `rank: float`
- [x] `AgentResourceResult` dataclass: trimmed shape from design spec §3.2 (id, title, summary, tags, top_level_categories, url)
- [x] Service is injectable via FastAPI dependency
