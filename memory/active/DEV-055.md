## Requirements

Enhance the Dashboard page to surface key resource statistics at a glance:

### Stats to display

1. **Resources in processing** — count of resources with status `processing` or `pending`
2. **Total resources added** — count of all resources for this user
3. **Categories identified** — count of unique tags/categories extracted by the LLM across all processed resources
4. **Category ranking** — ordered list of categories by number of associated resources (most → least), shown as a ranked list or bar chart

### Acceptance Criteria

- [ ] Dashboard shows a stats section with 4 metrics: "In Processing", "Total Added", "Categories Identified", "Top Categories"
- [ ] Stats are fetched from the backend (a new GET /dashboard/stats or derived from existing endpoints)
- [ ] "In Processing" count reflects resources with status in (`pending`, `processing`)
- [ ] "Total Added" count reflects all user resources regardless of status
- [ ] "Categories Identified" reflects distinct tag values across all user resources
- [ ] "Top Categories" shows a ranked list (category name + resource count), sorted descending
- [ ] Stats update on page load / refresh
- [ ] Empty state handled gracefully (0 values, no blank/error)

### Backend Notes

- May require a new GET /stats endpoint or extend GET /resources to support aggregation
- Category data comes from the `tags` field populated by DEV-022 (LLM processing — already complete)
- Stats should be scoped to the authenticated user

### Frontend Notes

- Design as stat cards (number + label) for the 4 metrics
- Category ranking can be a simple ordered list with counts, or a minimal bar chart
