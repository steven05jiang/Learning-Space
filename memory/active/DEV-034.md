## Requirements

Implement two read endpoints for chat history:

1. `GET /chat/conversations` — list the current user's conversations (paginated)
2. `GET /chat/conversations/{conversation_id}/messages` — get all messages for a specific conversation

### Context

- `models/conversation.py` — `Conversation` and `Message` ORM models (DEV-035, PR #117)
- `schemas/conversation.py` — `ConversationListResponse`, `ConversationWithMessagesResponse`, `ConversationResponse`, `MessageResponse` schemas already exist
- `routers/chat.py` — created by DEV-033 (PR #222); add these endpoints to the same file
- `main.py` — chat router already registered by DEV-033

### Endpoint 1: GET /chat/conversations

**Query params**:
- `limit: int = 20` (max 100)
- `offset: int = 0`

**Logic**:
1. Auth gate — `get_current_user` dependency
2. Query `conversations` table where `user_id == current_user.id`, ordered by `updated_at DESC`
3. Apply limit/offset pagination
4. Return `ConversationListResponse(items=[...], total=<total count>, limit=limit, offset=offset)`

### Endpoint 2: GET /chat/conversations/{conversation_id}/messages

**Path param**: `conversation_id: UUID`

**Logic**:
1. Auth gate — `get_current_user` dependency
2. Load conversation by `conversation_id`; 404 if not found, 403 if `conversation.user_id != current_user.id`
3. Load all messages for the conversation ordered by `created_at ASC`
4. Return `ConversationWithMessagesResponse` (id, user_id, title, created_at, updated_at, messages=[...])

### Acceptance criteria

- `GET /chat/conversations` → 200, returns paginated list of user's conversations (newest first)
- `GET /chat/conversations/{id}/messages` → 200, returns conversation + ordered messages
- Other user's conversation_id → 403
- Non-existent conversation_id → 404
- Unauthenticated → 401
- Empty conversations list → 200 with `items: [], total: 0`
