## Requirements

Implement `POST /graph/expand` — allows the frontend to drill down into a specific graph node and get its neighbors (child nodes + edges).

### Request body

```json
{ "node_id": "AI", "direction": "out" }
```

- `node_id` (required): the tag name to expand
- `direction` (optional): `"out"` (default), `"in"`, `"both"`

### Response shape (must match frontend mock contract)

```json
{
  "nodes": [
    { "id": "Python", "label": "Python", "level": "child" }
  ],
  "edges": [
    { "source": "AI", "target": "Python", "weight": 3 }
  ]
}
```

### Implementation notes

- Add route to `apps/api/routers/graph.py`
- Query Neo4j for direct neighbors of `node_id` scoped to `current_user.id`
- Return neighbors as `level: "child"` nodes + their edges to the root node
- If node not found or has no neighbors → return `{ "nodes": [], "edges": [] }`
- Reuse `GraphNode`, `GraphEdge`, `GraphResponse` schemas from DEV-028
- Add `get_neighbors(owner_id, node_id, direction)` method to `GraphService`

### Pydantic request schema

```python
class GraphExpandRequest(BaseModel):
    node_id: str
    direction: str = "out"  # "out", "in", "both"
```
