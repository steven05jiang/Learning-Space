## Requirements

**Type:** Backend
**BDD Reference:** All worker scenarios (INT-024–028)
**Description:** Orchestrate the full processing pipeline as an async worker job:
1. Set status → PROCESSING
2. Fetch content — use unauthenticated URL fetcher (DEV-020) for URL resources; use `original_content` directly for text resources. **DEV-021 (authenticated fetcher) is deferred — do not implement or require it.**
3. LLM processing (DEV-022) — extract title, summary, tags
4. Update resource in DB (status → READY, write title/summary/tags)
5. Trigger graph update (DEV-026 — will be integrated later; leave a clear hook/call site)
6. Handle errors at each step → status → FAILED with status_message

**Acceptance Criteria:**
- [ ] Pipeline runs end-to-end for URL resources (unauthenticated fetch only)
- [ ] Pipeline runs end-to-end for text resources (skip fetch step, use original_content)
- [ ] Status transitions: PENDING → PROCESSING → READY or FAILED
- [ ] LLM failure sets status=FAILED with descriptive status_message
- [ ] Fetch failure sets status=FAILED with descriptive status_message
- [ ] Graph update hook is present (can be a no-op / placeholder call for now; DEV-026 will wire it)
