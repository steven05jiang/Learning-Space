## Requirements

Add the `GET /api/v1/resources/search` FastAPI endpoint. Thin wrapper around `ResourceSearchService`. Validates query parameters (`q`, `tag`, `limit`, `offset`) via Pydantic, resolves `owner_id` from session, calls the service, and serializes the result as `ResourceSearchResponse`. Includes `rank` field in each resource object.

### Acceptance Criteria

- [ ] `GET /api/v1/resources/search?q=...` returns 200 + `{resources: [...], total: N}`
- [ ] `ResourceSearchRequest` Pydantic schema: `q` (required, 1–500 chars, strip_whitespace), `tag` (optional), `limit` (default 20, 1–100), `offset` (default 0)
- [ ] `ResourceSearchResponse` Pydantic schema: `resources: List[ResourceResponse]` (with `rank` field added), `total: int`
- [ ] Empty `q` → 400 `SEARCH_QUERY_EMPTY`
- [ ] `q` > 500 chars → 400 `SEARCH_QUERY_TOO_LONG`
- [ ] Unauthenticated → 401
- [ ] `limit` / `offset` out of range → 422 (FastAPI default)
- [ ] Endpoint registered in router under `/resources` prefix
- [ ] `rank` float field included in each resource in response
