## Requirements

Implement `GET /graph` (and `GET /graph?root=TagName`) — the core knowledge graph exploration endpoint. Returns nodes and edges for the authenticated user's personal knowledge graph from Neo4j.

### Response shape (must match frontend mock contract)

```json
{
  "nodes": [
    { "id": "AI", "label": "AI", "level": "current" }
  ],
  "edges": [
    { "source": "AI", "target": "Python", "weight": 3 }
  ]
}
```

- `level` values: `"root"`, `"current"`, `"child"` (three-level display per technical design)
- `id` and `label` = tag name (node_id for graph operations)
- Without `?root=TagName`: return all tag nodes + edges for the user
- With `?root=TagName`: return the given tag as `level: "current"`, its direct neighbors as `level: "child"`, and the neighbors' neighbors as `level: "parent"`

### Implementation notes

- Add route to `apps/api/routers/graph.py`
- Query Neo4j via `graph_service` (or direct neo4j_driver session)
- Use `owner_id = current_user.id` to scope to the authenticated user
- Add Pydantic schemas for request/response to `apps/api/schemas/` (GraphNode, GraphEdge, GraphResponse)
- Register router already exists at `/graph` prefix in main app
- Must handle empty graph (no tags yet) → return `{ "nodes": [], "edges": [] }`

### Schema to add (in `apps/api/schemas/graph.py` — create if not exists)

```python
class GraphNode(BaseModel):
    id: str
    label: str
    level: str  # "root", "current", "child"

class GraphEdge(BaseModel):
    source: str
    target: str
    weight: int

class GraphResponse(BaseModel):
    nodes: list[GraphNode]
    edges: list[GraphEdge]
```
