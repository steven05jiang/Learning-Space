## Requirements

Add pytest unit tests (no real DB) for `ResourceSearchService` and the `GET /resources/search` endpoint. Uses mocked `AsyncSession` and mocked service responses. Covers critical paths: successful search, tag filtering, READY-only enforcement, pagination, and validation error codes.

**NOTE:** DEV-073 already created `apps/api/tests/test_resource_search_service.py` (9 tests for the service) and DEV-074 already created `apps/api/tests/test_resources_search.py` (9 tests for the endpoint). Check if these already cover the acceptance criteria below before adding more tests. If coverage is sufficient, this task may just be a verification pass + any gap tests.

### Acceptance Criteria

- [ ] `pytest -m "not integration"` passes with all search tests included
- [ ] Test: keyword match returns ranked results (rank DESC order)
- [ ] Test: `tag=` filter param is passed to service; results narrowed
- [ ] Test: non-READY resources not returned (READY-only filter)
- [ ] Test: `q=""` → 400 `SEARCH_QUERY_EMPTY`
- [ ] Test: `q` > 500 chars → 400 `SEARCH_QUERY_TOO_LONG`
- [ ] Test: unauthenticated request → 401
- [ ] Test: `rank` field present in each result
- [ ] Test: `total` reflects full match count, not page size
