# BUG-007: Settings linked accounts shows incorrect connection status

**Status:** ⏳ Pending
**Priority:** P1 — High
**Reported:** 2026-03-17
**Branch:** (pending)
**PR:** (pending)

## Description

On the Settings page (Account Management / Linked Accounts section), even when the user is already logged in via Google OAuth, the UI incorrectly shows the Google account as "not connected" and prompts the user to connect. The linked account status does not reflect the actual OAuth session state.

## Steps to Reproduce

1. Log in using Google OAuth
2. Navigate to Settings → Account Management
3. Observe that the "Linked Accounts" panel shows Google as not connected and shows a "Connect" button

## Expected Behavior

Since the user is already authenticated via Google OAuth, the Google entry in Linked Accounts should show as connected (with a "Disconnect" option, not "Connect").

## Root Cause Hypothesis

The Settings UI likely queries the linked accounts data from the API (GET /auth/accounts or similar) and that endpoint may return no linked accounts — possibly because the OAuth login flow (DEV-005) does not persist the provider association into the accounts table upon login. Related to BUG-008 (local account not created on OAuth login).

## Acceptance Criteria

- [ ] After logging in via Google, Settings page shows Google as "Connected"
- [ ] "Disconnect" option is shown (not "Connect") for the authenticated provider
- [ ] If a user has not connected a provider, "Connect" is shown correctly
