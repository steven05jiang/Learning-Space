# DEV-067: Disable search button — "coming soon" tooltip

**Status:** ⏳ Pending
**Priority:** High
**Started:** (pending)
**Branch:** (pending)
**PR:** (pending)

## Requirements

The Search nav item in the app sidebar should appear disabled with a tooltip explaining
the feature is not yet available.

### Key file: `apps/web/components/app-sidebar.tsx`

Current state: `navItems` array includes `{ title: "Search", href: "/search", icon: Search }`
rendered as a `SidebarMenuButton asChild` wrapping a `<Link>`.

Target state:
- Search item is visually dimmed (e.g. `opacity-50 cursor-not-allowed pointer-events-none` or similar)
- Clicking it does nothing (no navigation to /search)
- A tooltip says "Search coming soon" on hover

Implementation approach:
- Remove Search from navItems (or add a `disabled` flag)
- Render the Search item separately as a non-link button with `disabled` styling
- Use the existing `tooltip` prop on `SidebarMenuButton` for the "Search coming soon" text
- The SidebarMenuButton's `tooltip` prop already handles collapsed-sidebar tooltip display — reuse it

### Acceptance criteria
- Search nav item renders in sidebar but is visually disabled
- Clicking produces no navigation
- Hovering (or in collapsed mode) shows "Search coming soon" tooltip
- `make web-lint` passes
- `make web-build` passes
