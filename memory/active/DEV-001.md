## Requirements

Initialize the monorepo layout and scaffold both applications so that all subsequent tasks have a consistent project structure to build on.

**Deliverables:**
- Monorepo root with a clear directory layout: `apps/api/` (FastAPI backend) and `apps/web/` (Next.js frontend)
- `apps/api/`: FastAPI app scaffolded with `uv init`, directory structure for `routers/`, `models/`, `services/`, `workers/`, `core/` (config, deps, errors)
- `apps/web/`: Next.js app scaffolded (TypeScript, App Router), directory structure for `app/`, `components/`, `lib/`
- Root `docker-compose.yml` with services: `api`, `web`, `postgres`, `neo4j`, `redis` (for task queue)
- Root `.github/workflows/ci.yml`: runs lint + test for both apps on push/PR
- `apps/api/pyproject.toml` managed by `uv`; `apps/web/package.json`
- `.gitignore` covering Python, Node, env files, and IDE artifacts
- `README.md` at repo root with a one-paragraph project description and local dev quickstart

**Tech stack (from technical design):**
- Backend: Python 3.12+, FastAPI, SQLAlchemy 2.x, Alembic, `uv` for dependency management
- Frontend: Next.js 14+ (App Router), TypeScript, Tailwind CSS
- Infra: PostgreSQL 16, Neo4j 5, Redis 7
- CI: GitHub Actions

**Exit criteria (Tier 1 partial):**
- `cd apps/api && uv run uvicorn main:app` starts without import errors
- `cd apps/web && npm run dev` starts without errors
- CI workflow file is valid YAML and references correct paths
- Directory structure matches the layout above
