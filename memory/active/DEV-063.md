## Requirements

Add a "Categories" section to the Settings page. Users can see all root categories (system and their own user-created ones), add custom categories, and delete their own (non-system) categories.

### Acceptance Criteria

- [ ] Settings page has a "Categories" section (below or alongside the linked accounts section)
- [ ] System categories listed with a lock icon; no delete button shown
- [ ] User-created categories listed with a delete (×) button
- [ ] Clicking delete shows a confirmation; on confirm calls DELETE /categories/{id}
- [ ] "Add category" input field + submit button; calls POST /categories; shows validation error on duplicate name
- [ ] Category list refreshes after successful add or delete (no full page reload)
- [ ] Empty state shown when user has no custom categories
