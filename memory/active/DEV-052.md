## Requirements

Connect the existing graph visualization UI (already built with mock data in DEV-044/045) to the live backend APIs. The UI shell is complete — this task replaces mock data with real API calls.

### Frontend component

- `apps/web/components/knowledge-graph.tsx` — currently uses hardcoded `knowledgeData` mock
- `apps/web/app/knowledge-graph/page.tsx` — the page that renders the component

### APIs to wire

| Endpoint | When | What to do with response |
|----------|------|--------------------------|
| `GET /api/graph` | On mount | Populate initial graph nodes + edges |
| `POST /api/graph/expand` | On node click (expand) | Add new child nodes + edges to graph |
| `GET /api/graph/nodes/{id}/resources` | On node click (panel) | Show resources in the resource panel dialog |

### Backend response shapes (live)

**GET /api/graph:**
```json
{ "nodes": [{"id": "AI", "label": "AI", "level": "current"}], "edges": [{"source": "AI", "target": "Python", "weight": 3}] }
```

**POST /api/graph/expand:**
```json
{ "nodes": [{"id": "Python", "label": "Python", "level": "child"}], "edges": [{"source": "AI", "target": "Python", "weight": 3}] }
```

**GET /api/graph/nodes/{id}/resources:**
```json
{ "items": [{"id": "uuid", "title": "...", "summary": "...", "status": "READY", "tags": [...]}], "total": N }
```

### Implementation approach

1. Remove the hardcoded `knowledgeData` mock from `knowledge-graph.tsx`
2. Add `useEffect` to fetch `GET /api/graph` on mount; set graph data from response
3. On node click: call `POST /api/graph/expand` with `{node_id: node.id}` and merge new nodes/edges into graph state
4. On node click: call `GET /api/graph/nodes/{node.id}/resources` and display in the dialog panel (already implemented as ResourcePanel dialog)
5. Handle loading state (show spinner while fetching)
6. Handle empty graph (show "No knowledge graph yet — add and process some resources" message)
7. Handle API errors gracefully (show error toast or fallback message)

### Node/Edge type mapping

The existing `KnowledgeNode` type uses `name`, `category`, `color`, `val`, `resources`. Map from API:
- `node.id` → `id`
- `node.label` → `name`
- `node.level` → `category` (or use for styling)
- Color by level: `"current"` → blue, `"child"` → green, `"parent"` → gray, `"root"` → blue
- `val` = node weight/importance (can default to 10)

The existing `KnowledgeLink` uses `source`, `target`, `relationship`. Map from API:
- `edge.source` → `source`
- `edge.target` → `target`
- `edge.weight` → store as `relationship` label or separate field
