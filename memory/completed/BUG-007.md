# BUG-007: Settings linked accounts shows incorrect connection status

**Status:** ✅ Completed (resolved by BUG-008)

**Reported:** 2026-03-17
**Completed:** 2026-03-18
**Branch:** fix/bug-008-oauth-local-account
**PR:** #65 (merged)

## Description

Settings page showed Google as "not connected" even after OAuth login, prompting user to connect.

## Root Cause

OAuth login did not persist the provider association into the `accounts` table, so the linked accounts query returned empty.

## Resolution

Resolved as part of BUG-008 (PR #65): OAuth callback now creates/upserts a row in the `accounts` table on every login, and GET /auth/accounts endpoint was added. Settings page linked accounts now reflects the correct connected state.
