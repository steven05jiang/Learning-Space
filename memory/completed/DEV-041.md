## Requirements

Implement the resource submission form in the Next.js frontend (`apps/web`). Authenticated users can submit a URL (and optionally a note) to add a new resource. The form calls `POST /resources` on the backend.

**Dependencies satisfied:** DEV-039 (OAuth login UI) ✅, DEV-013 (POST /resources backend) ✅

**Acceptance criteria:**

- A submission form exists (on `/resources/new` or inline on `/resources` or dashboard)
- Form has a URL input field (required) and an optional note/description field
- On submit, calls `POST /resources` with the user's auth cookie/token
- Shows a loading state while submitting
- On success: redirects to resource list or shows a success message
- On error: shows a user-friendly error message
- Redirects to login if the user is unauthenticated
