# DEV-027: Graph sync job for resource deletion

**Status:** ✅ Completed
**Priority:** High
**Started:** 2026-03-21
**Branch:** feature/dev-027-graph-sync-deletion
**PR:** #103

## Requirements

When a resource is deleted, the knowledge graph must be updated to decrement/remove the tag relationships that were backed by that resource. This ensures graph consistency when users delete resources.

### What to implement

1. **Update the `sync_graph` stub in `apps/api/workers/tasks.py`** — replace the stub with real logic that calls `graph_service.remove_resource_tags(owner_id, tags)` and `graph_service.cleanup_orphan_tags(owner_id)` for the "delete" operation.

2. **Hook deletion into the resource DELETE endpoint** — in `apps/api/routers/resources.py`, after a resource is deleted from PostgreSQL, enqueue a `sync_graph` job (with `operation="delete"`, passing the resource's `owner_id` and `tags`) via the queue service.

   Current delete endpoint likely just removes the DB row. It needs to also enqueue a graph sync job.

3. **Unit tests** — add tests for:
   - `sync_graph` with `operation="delete"` calls `graph_service.remove_resource_tags` and `cleanup_orphan_tags`
   - DELETE /resources/{id} enqueues a graph sync job after deletion

### Key files

- `apps/api/workers/tasks.py` — `sync_graph()` stub to implement
- `apps/api/routers/resources.py` — DELETE endpoint to update
- `apps/api/services/graph_service.py` — `remove_resource_tags()` and `cleanup_orphan_tags()` already implemented (DEV-025)
- `apps/api/tests/test_jobs_api.py` or `tests/test_resources_crud.py` — unit tests

### Graph service methods available

- `graph_service.remove_resource_tags(owner_id: int, old_tags: List[str])` — decrements weights, removes zero-weight edges
- `graph_service.cleanup_orphan_tags(owner_id: int)` — removes Tag nodes with no remaining edges

### Important

- Deletion should pass the resource's `tags` to `remove_resource_tags` — need to capture tags BEFORE deleting the row from PostgreSQL (or read from the resource object before deletion)
- Graph sync failure should NOT block the delete response (non-blocking, same pattern as graph update in process_resource)
- Run `make api-lint` and `make api-test` — both must pass
