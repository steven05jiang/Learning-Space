# DEV-054: Duplicate URL detection with user-facing notification

## Requirements

When a user attempts to submit a URL that already exists in their resource list, the system should:
1. **Backend**: Detect the duplicate before creating a new resource. Return a clear error (e.g. HTTP 409 Conflict) with a message indicating the resource already exists.
2. **Frontend**: Display a user-friendly toast/inline error: "This resource has already been added."

### Acceptance Criteria

- [x] POST /resources returns HTTP 409 with `{"detail": "This resource has already been added."}` if the URL already exists for this user
- [x] The resource submission form catches the 409 response and displays the message "This resource has already been added." to the user (toast or inline error)
- [x] The duplicate check is scoped per-user (two different users can submit the same URL)
- [x] Existing resources are not modified or duplicated
- [x] Unit test covers the duplicate URL scenario
