## Requirements

Add an Alembic migration that:
1. Enables the `pgvector` extension (`CREATE EXTENSION IF NOT EXISTS vector`)
2. Creates the `resource_embeddings` table
3. Creates an IVFFlat index for approximate nearest-neighbor search

**Embedding dimension: 2048** (Qwen/Qwen3-Embedding-4B via SiliconFlow — NOT the 1536 default in design-search.md §5.1)

### Schema

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE resource_embeddings (
    resource_id INTEGER PRIMARY KEY REFERENCES resources(id) ON DELETE CASCADE,
    embedding   vector(2048) NOT NULL,
    model       VARCHAR(100) NOT NULL,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX resource_embeddings_vec_idx
    ON resource_embeddings
    USING ivfflat (embedding vector_cosine_ops)
    WITH (lists = 100);
```

Notes:
- `resource_id` type should match `resources.id` type (check if INTEGER or UUID in current schema)
- `ON DELETE CASCADE` handles embedding cleanup automatically when resource is deleted
- IVFFlat `lists = 100` is appropriate for up to ~10K resources
- Use `op.execute()` with `autocommit_block()` if index creation requires non-transactional DDL (same pattern as DEV-072 GIN index fix)
