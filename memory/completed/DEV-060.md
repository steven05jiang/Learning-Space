## Requirements

Create the PostgreSQL `categories` table for root-level topic groupings. Seed 10 system categories on migration. Expose GET/POST/DELETE /categories endpoints so the frontend and LLM pipeline can read and manage categories.

Seeded system categories (is_system=true): Technology, Science, Business, Arts, Health, Education, Politics, Entertainment, Sports, Philosophy

### Acceptance Criteria

- [x] `categories` table created via Alembic migration: columns `id`, `name`, `is_system` (bool), `owner_id` (nullable FK to users; null for system categories), `created_at`
- [x] 10 system categories seeded in migration (idempotent — safe to re-run)
- [x] `GET /categories` returns all categories visible to the user (all system + own user-created); no auth required to list system categories
- [x] `POST /categories` creates a user-owned category; name must be unique per user; returns 409 on duplicate
- [x] `DELETE /categories/{id}` deletes a user-owned category; returns 403 if attempting to delete a system category; returns 404 if not found or not owned by user
- [x] Unit tests for all three endpoints (success paths + error cases)

## Implementation Summary

Categories table with Alembic migration seeding 10 system categories (idempotent, parameterized SQL to avoid injection). GET/POST/DELETE /categories endpoints with proper auth/authz, user isolation, and 403/404 distinction. Fixed SQL injection (f-string → text().bindparams), trailing slash routing, and test database seeding. 19 unit tests pass.
