## Requirements

Implement the OAuth login page for the Next.js frontend (`apps/web`). Users must be able to initiate an OAuth login flow with at least one provider (e.g. Google, GitHub, Twitter/X). The page should present login buttons per provider, redirect to the backend OAuth endpoint (`/auth/login/{provider}`), and handle the callback redirect back to the app.

**Dependencies satisfied:** DEV-005 (OAuth login flow) ✅, DEV-009 (GET /auth/me) ✅

**Acceptance criteria:**
- A `/login` page exists in the Next.js app with OAuth provider button(s)
- Clicking a provider button redirects to the backend `/auth/login/{provider}` endpoint
- After successful OAuth callback, the user is redirected to the app (e.g. `/dashboard` or `/resources`)
- Auth state is stored (e.g. cookie or localStorage via the /auth/me endpoint response)
- If the user is already logged in, `/login` redirects away
- Page is functional and styled simply (no need for polish)
