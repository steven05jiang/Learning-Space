## Requirements

Deploy the API and worker services to Railway using the Dockerfile created in DEV-047 (PR #165).

### Scope

1. **Railway project setup** — create a new Railway project (or use the template created in OPS-002 Step 5); add two services: `api` and `worker`
2. **Dockerfile configuration** — both services use `apps/api/Dockerfile`; `api` starts with `CMD ["uvicorn", ...]`, `worker` with `CMD ["arq", ...]` (multi-stage or build args to select entrypoint)
3. **Environment variables** — set all vars from `apps/api/.env.production.example` in Railway dashboard for both services:
   - `DATABASE_URL` (Supabase)
   - `NEO4J_URI`, `NEO4J_USERNAME`, `NEO4J_PASSWORD` (AuraDB)
   - `REDIS_URL` (Upstash)
   - `SECRET_KEY`, `ALLOWED_EMAILS`, `LLM_PROVIDER`, `ANTHROPIC_API_KEY` (or equivalent)
   - `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`
   - `TWITTER_CLIENT_ID`, `TWITTER_CLIENT_SECRET` (if needed)
   - `FRONTEND_URL` (to be updated after OPS-004)
4. **Alembic migrations** — configure a Railway deploy hook or start command to run `alembic upgrade head` before the API server starts
5. **Auto-deploy** — connect Railway project to the GitHub repo; trigger deploy on push to `main`
6. **Health check** — verify Railway health check hits `GET /health` → 200 after deploy
7. **Railway URL** — note the generated `*.railway.app` URL for the API; update `memory/infra.md`

### Acceptance Criteria

- `GET https://<api>.railway.app/health` returns `{"status": "ok"}`
- Worker service is running (Railway dashboard shows it active)
- Alembic migrations ran successfully (visible in Railway deploy logs)
- Auto-deploy from `main` is confirmed working
- `memory/infra.md` updated with Railway service URLs

### Key Files

- `apps/api/Dockerfile` — already created (DEV-047, PR #165)
- `apps/api/.env.production.example` — template for Railway env vars
- `apps/api/alembic.ini` + `apps/api/alembic/` — migration config
- `memory/infra.md` — update with Railway URLs after deploy

### Dependencies

- DEV-047 ✅ (Dockerfile exists)
- OPS-002 ✅ (Supabase, Neo4j AuraDB, Upstash all provisioned; credentials in apps/api/.env.production)
