## Requirements

Update the `process_resource` worker job to respect the `processing_status` field. Workers must set the status to `processing` when starting, `success` or `failed` on completion, and skip resources that are already in a terminal state (`success` or `failed`). No automatic re-processing on failure ‚Äî recovery is manual (see DEV-059).

### Acceptance Criteria

- [x] Worker sets `processing_status = failed` (with `status_message`) on any error
- [x] Worker skips (logs and returns early) if `processing_status` is already `success` or `failed` when job starts
