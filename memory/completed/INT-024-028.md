### INT-024: Worker processes URL resource successfully
- Create a PENDING resource with type=URL and a test URL
- Mock the URL fetcher to return HTML content (use `respx` or `unittest.mock`)
- Mock the LLM to return deterministic title/summary/tags
- Run the process_resource worker job directly (call the task function, don't use Celery)
- Assert resource status → READY, title/summary/tags populated, graph update called

### INT-025: Worker processes text resource successfully
- Create a PENDING resource with type=TEXT and original_content set
- Mock the LLM to return deterministic title/summary/tags (no URL fetch needed)
- Run the process_resource worker job
- Assert resource status → READY, title/summary/tags populated

### INT-026: URL requires login, user has linked account — provider fetch succeeds
- Create a PENDING URL resource where the URL returns 401 on plain fetch
- User has a linked Twitter account with a valid token
- Mock provider fetch (Twitter API) to return tweet content
- Run the worker job
- Assert resource status → READY, content extracted via provider

### INT-027: URL requires login, user has no linked account — FAILED
- Create a PENDING URL resource where the URL returns 401 on plain fetch
- User has NO linked account for the provider
- Run the worker job
- Assert resource status → FAILED, status_message explains auth required

### INT-028: LLM processing fails — FAILED
- Create a PENDING resource (URL or text)
- Mock fetch to succeed (or use text resource)
- Mock LLM to raise an exception
- Run the worker job
- Assert resource status → FAILED, status_message set
