# BUG-002: POST /resources returns 500 — datetime timezone mismatch

## Problem
`apps/api/routers/resources.py` lines 64–65 explicitly pass `datetime.now(timezone.utc)` (timezone-aware) to `created_at` and `updated_at` columns declared as `Column(DateTime)` — which maps to `TIMESTAMP WITHOUT TIME ZONE` in PostgreSQL. asyncpg raises a DataError.

## Implementation Summary
Removed explicit `created_at` and `updated_at` arguments from the `Resource()` constructor in `create_resource()`. ORM defaults (`default=datetime.utcnow`) handle timestamp generation correctly. Removed unused `timezone` import. All tests pass.
