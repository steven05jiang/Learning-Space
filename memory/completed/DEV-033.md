## Requirements

Implement `POST /chat` ‚Äî a stateful chat endpoint that creates or continues a conversation,
persists both the user message and the assistant response to the database, and delegates to
the existing `AgentService`.

### Context

- `models/conversation.py` ‚Äî `Conversation` and `Message` ORM models already exist (DEV-035, PR #117)
- `schemas/conversation.py` ‚Äî `ConversationResponse`, `MessageResponse`, `ConversationWithMessagesResponse` schemas already exist
- `services/agent_service.py` ‚Äî `AgentService.query(user, AgentQuery)` already exists; accepts `conversation_history: List[ConversationMessage]`
- `routers/agent.py` ‚Äî existing stateless `/agent/query` endpoint (no persistence); leave it in place
- `main.py` ‚Äî must register the new chat router

### Endpoint: POST /chat

**Request schema** (`ChatRequest`, new in `schemas/chat.py`):
```python
class ChatRequest(BaseModel):
    message: str          # the user's message (non-empty, max 2000 chars)
    conversation_id: Optional[UUID] = None  # None = create new conversation
```

**Response schema** (`ChatResponse`, new in `schemas/chat.py`):
```python
class ChatResponse(BaseModel):
    conversation_id: UUID
    message_id: UUID      # UUID of the persisted assistant Message row
    response: str         # agent's text response
    role: str = "assistant"
```

**Logic**:
1. Auth gate ‚Äî `get_current_user` dependency
2. If `conversation_id` provided: load from DB, verify `conversation.user_id == current_user.id` (404 if not found, 403 if wrong user)
3. If no `conversation_id`: create a new `Conversation` row (title=None initially)
4. Persist the user's `Message` row (`role=MessageRole.USER, content=request.message`)
5. Load full message history for this conversation from DB (ordered by `created_at ASC`)
6. Build `AgentQuery(query=request.message, conversation_history=[...])` from all prior messages (exclude the just-inserted user message to avoid duplication in the history list passed to agent ‚Äî the query itself is the current turn)
7. Call `await agent_service.query(current_user, agent_query)` ‚Üí `AgentResponse`
8. Persist the assistant `Message` row (`role=MessageRole.ASSISTANT, content=response.response`)
9. Update `conversation.updated_at = datetime.utcnow()`
10. Return `ChatResponse(conversation_id=conversation.id, message_id=<assistant message id>, response=response.response)`

**Router**: new file `routers/chat.py`, prefix `/chat`, tag `chat`

**Register in `main.py`**: `app.include_router(chat.router)` (alongside existing routers)

### Acceptance criteria

- `POST /chat` with no `conversation_id` ‚Üí 200, creates new conversation, returns `conversation_id`
- `POST /chat` with existing `conversation_id` ‚Üí 200, continues conversation, history passed to agent
- Unauthenticated request ‚Üí 401
- Wrong user accesses another user's conversation ‚Üí 403
- Non-existent `conversation_id` ‚Üí 404
- Empty message ‚Üí 422 (Pydantic validation)
- User message and assistant response both persisted in `messages` table
- `conversation.updated_at` updated on each message
