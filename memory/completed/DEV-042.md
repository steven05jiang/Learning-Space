## Requirements

Implement the resource list page in the Next.js frontend (`apps/web`). Authenticated users can see all their submitted resources with their processing status and metadata.

## Implementation Summary

Implemented `/resources` page with resource cards showing URL/title, status badges (PENDING/PROCESSING/READY/FAILED), summary and tags for READY resources, and empty state with CTA. Polls every 10 seconds for live status updates. XSS protection via `getSafeUrl()` helper validating http/https-only URLs. Env var fallback to localhost:8000. Redirects unauthenticated users to login. Fixed useEffect dependency array to prevent double-fetch cycle.
