# OPS-002: Provision Cloud Data Services

**Status:** ✅ Complete
**Priority:** High

## Overview

Provision the three managed data services needed for production deployment:
1. **Supabase** — PostgreSQL database
2. **Neo4j AuraDB Free** — Graph database
3. **Upstash** — Redis (task queue)

All three have free tiers sufficient for initial user-feedback stage.

---

## Step 1 — Supabase (PostgreSQL)

1. Go to https://supabase.com → "Start your project" → sign up/log in
2. Create a new project:
   - **Name:** `learning-space`
   - **Database Password:** generate a strong password and save it
   - **Region:** pick closest to your Railway region (e.g. US East)
3. Wait for project to provision (~2 min)
4. Go to **Project Settings → Database**
5. Under **Connection string**, select **URI** tab, copy the string:
   `postgresql://postgres:[PASSWORD]@db.[PROJECT-REF].supabase.co:5432/postgres`
6. Save as `DATABASE_URL` (replace `[PASSWORD]` with your actual password)

> Note: Use `postgresql+asyncpg://...` (swap `postgresql://` → `postgresql+asyncpg://`) for the API.

---

## Step 2 — Neo4j AuraDB Free

1. Go to https://console.neo4j.io → sign up/log in
2. Click **New Instance** → choose **AuraDB Free**
3. Note the generated **password** immediately (shown once) → save it
4. Wait for instance to start (~3 min)
5. Once running, copy:
   - **Connection URI:** `neo4j+s://xxxxxxxx.databases.neo4j.io`
   - **Username:** `neo4j`
   - **Password:** (saved above)
6. Save as:
   - `NEO4J_URI=neo4j+s://xxxxxxxx.databases.neo4j.io`
   - `NEO4J_USERNAME=neo4j`
   - `NEO4J_PASSWORD=<saved password>`

---

## Step 3 — Upstash Redis

1. Go to https://upstash.com → sign up/log in
2. Click **Create Database**
   - **Name:** `learning-space`
   - **Type:** Regional
   - **Region:** pick same region as Railway
   - **Eviction:** disabled (task queue needs persistence)
3. Once created, go to **Details** tab
4. Copy the **Redis URL** under "Connect":
   `redis://:PASSWORD@HOST:PORT`
   or the `rediss://` (TLS) variant — prefer TLS for production
5. Save as `REDIS_URL=rediss://:PASSWORD@HOST:PORT`

---

## Step 4 — Document in memory/infra.md

Once all three are provisioned, update `memory/infra.md` with:
- Service names, regions, free tier limits
- Environment variable names (NOT values) for each service
- Links to dashboards

---

## Step 5 — Prepare .env.production template

Create `apps/api/.env.production.example` with all required env vars:
```
DATABASE_URL=postgresql+asyncpg://postgres:PASSWORD@db.PROJECT.supabase.co:5432/postgres
NEO4J_URI=neo4j+s://XXXXXXXX.databases.neo4j.io
NEO4J_USERNAME=neo4j
NEO4J_PASSWORD=
REDIS_URL=rediss://:PASSWORD@HOST:PORT
JWT_SECRET_KEY=
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
ALLOWED_EMAILS=
LLM_PROVIDER=groq
GROQ_API_KEY=
ANTHROPIC_API_KEY=
OAUTH_REDIRECT_BASE_URL=https://your-railway-domain.railway.app
```
