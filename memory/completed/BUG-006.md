# BUG-006: Pending tag overflow in resources page

**Status:** ✅ Completed
**Priority:** P2 — Medium
**Reported:** 2026-03-17

**PR:** #63

## Description

On the resources list page, the "Pending" status badge/tag overflows its container. The tag text or pill extends beyond its expected bounds, breaking the layout.

## Steps to Reproduce

1. Log in and navigate to the Resources page
2. Observe any resource with "Pending" processing status
3. The status tag overflows its container

## Expected Behavior

"Pending" tag fits within its badge/pill container, consistent with other status tags.

## Acceptance Criteria

- [x] Pending status tag renders within its container without overflow
- [x] Fix applies to all status variants (Pending, Processing, Done, Failed)
- [x] No regression on other resource list UI elements
