# BUG-006: Pending tag overflow in resources page

**Status:** ✅ Completed

## Description

Resources list page: when a resource has a long URL title, the "Pending" tag overflows out of the card bounds instead of staying contained within its row.

## Root Cause

The title element lacked flex constraints to prevent overflow. Without `min-w-0` and `flex-1`, flex children can exceed container bounds when content is too wide.

## Resolution

Applied `min-w-0 flex-1` classes to the title element (`<h3>`) in the resource card component. This allows the title to shrink below its intrinsic content width and prevents the pending tag from overflowing.
