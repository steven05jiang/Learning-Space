## Requirements

BDD integration test for search input validation edge cases. Verifies that the endpoint rejects empty queries and queries exceeding the 500-character limit with the correct error codes.

## Implementation Summary

Added 3 validation tests to `apps/api/tests/integration/test_int_search.py`: empty query → 400 SEARCH_QUERY_EMPTY, overlong query (501+ chars) → 400 SEARCH_QUERY_TOO_LONG, missing q param → 422. Also fixed a bug in the endpoint: removed `min_length=1` from `q: str = Query(...)` so whitespace-only queries reach the custom handler and return 400 (not 422 from FastAPI's built-in validator).
