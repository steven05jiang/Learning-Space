## Requirements

Implement `GET /graph` (and `GET /graph?root=TagName`) — the core knowledge graph exploration endpoint. Returns nodes and edges for the authenticated user's personal knowledge graph from Neo4j.

### Response shape (must match frontend mock contract)

```json
{
  "nodes": [
    { "id": "AI", "label": "AI", "level": "current" }
  ],
  "edges": [
    { "source": "AI", "target": "Python", "weight": 3 }
  ]
}
```

- `level` values: `"root"`, `"current"`, `"child"`, `"parent"` (three-level display per technical design)
- `id` and `label` = tag name (node_id for graph operations)
- Without `?root=TagName`: return all tag nodes + edges for the user
- With `?root=TagName`: return the given tag as `level: "current"`, its direct neighbors as `level: "child"`, and the neighbors' neighbors as `level: "parent"`

## Implementation Summary

Added `GraphNode`, `GraphEdge`, `GraphResponse` schemas in `schemas/graph.py`. Added `get_graph(owner_id, root=None)` method to `GraphService` with Neo4j queries supporting both all-nodes and rooted subgraph modes. Rooted subgraph fetches three levels (current → children → parents via Cypher) with edge deduplication. Added `GET /graph` route with optional `?root` parameter. 5 comprehensive unit tests with mocked Neo4j driver covering all levels and edge cases. Required 3 review rounds — rounds 1–2 fixed missing parent-level nodes and schema comment; approved on round 3.
