## Requirements

Add a `processing_status` enum field to the `resources` table to explicitly track whether a resource has been processed. This replaces implicit status inference and enables the worker to skip already-processed resources and users to manually trigger reprocessing.

Valid states: `pending` → `processing` → `success` | `failed`

### Acceptance Criteria

- [x] `processing_status` enum column added to `resources` table (values: `pending`, `processing`, `success`, `failed`; default: `pending`)
- [x] Alembic migration created; applies cleanly on fresh and existing databases
- [x] Existing rows backfilled: status=`READY` → `success`, status=`FAILED` → `failed`, all others → `pending`
- [x] Pydantic `Resource` response schema exposes `processing_status`
- [x] Unit test: migration idempotency and backfill correctness
