## Requirements

Add a tag editor to the resource detail page so users can add and remove tags after a resource is processed. Saving the updated tag set calls PATCH /resources/{id} with the new tags array and enqueues a graph resync job for that resource to keep edges consistent.

### Acceptance Criteria

- [x] Resource detail page displays current tags as chips with a × remove button on each
- [x] Tag input field allows adding new tags (confirmed with Enter or comma)
- [x] "Save tags" button calls `PATCH /resources/{id}` with the updated `tags` array
- [x] Backend enqueues a graph resync job on tag change (update Neo4j edges to reflect new tag set)
- [x] At least one root-level category tag must remain after editing; UI shows validation error if the last root category tag is removed
- [x] Toast confirmation shown on successful save
- [x] Save button disabled when no changes have been made

## Implementation Summary

Added tag editor to resource detail page with editable chips (×), Enter/comma input, save/cancel buttons. Backend PATCH /resources/{id} now enqueues graph_sync job when tags change. Categories fetched dynamically from /categories API for root category validation. 8 frontend tests + backend test for graph sync enqueueing.
