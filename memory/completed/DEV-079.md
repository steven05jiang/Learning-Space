## Requirements

Extend the worker pipeline to generate and store an embedding for each resource after the LLM step completes. Uses SiliconFlow's `Qwen/Qwen3-Embedding-4B` model (2048 dims) via the existing SiliconFlow API client pattern.

### New function: build_embedding_text()

```python
def build_embedding_text(resource: Resource) -> str:
    """Concatenate searchable fields for embedding generation."""
    parts = []
    if resource.title:
        parts.append(resource.title)
    if resource.summary:
        parts.append(resource.summary)
    if resource.tags:
        parts.append(" ".join(resource.tags))
    if resource.top_level_categories:
        parts.append(" ".join(resource.top_level_categories))
    return " ".join(parts)
```

### Worker pipeline change

After the existing LLM step (title/summary/tags written to DB), add:

```
fetch → LLM (title, summary, tags, categories) → DB update
  → [generate embedding → upsert resource_embeddings]   ← NEW
  → Neo4j graph update
```

```python
# After DB update, before Neo4j sync
embedding_text = build_embedding_text(resource)
embedding_vector = await embedding_client.embed(embedding_text)
await upsert_resource_embedding(session, resource.id, embedding_vector)
```

### SiliconFlow embedding client

Follow the same pattern as the existing SiliconFlow LLM client. Key details:
- Model: `Qwen/Qwen3-Embedding-4B`
- API endpoint: SiliconFlow embeddings endpoint (OpenAI-compatible: `POST /v1/embeddings`)
- Auth: `SILICONFLOW_API_KEY` (same key as LLM — no new env var needed)
- New env vars:
  - `EMBEDDING_MODEL` — default `Qwen/Qwen3-Embedding-4B`
  - `EMBEDDING_DIMENSIONS` — default `2048`
- Response: extract `data[0].embedding` (list of floats, length 2048)
