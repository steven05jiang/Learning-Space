## Scope

Write Layer 1 API integration tests for two groups:

### Group: health (CI: every PR)
- **INT-001** — GET /health → 200 OK; body contains `{"status": "ok"}` or similar
- **INT-002** — Any endpoint hit with a bad request returns the standard error response schema (check error format matches `{"detail": ..., "error_code": ...}` pattern from DEV-038)

### Group: auth (CI: every PR)
- **INT-003** — POST to `/auth/callback/twitter` with mock OAuth → creates new User + Account row; returns JWT/session
- **INT-004** — Same Twitter login again (existing user) → returns existing user (no new row)
- **INT-005** — Login with Google (different provider, same email) → links to existing user
- **INT-006** — Request to a protected endpoint (e.g. GET /auth/me) without a token → 401
- **INT-007** — Request with an expired/invalid JWT → 401
- **INT-008** — Authenticated user hits GET /auth/link/twitter → links new Account row to existing user
- **INT-009** — Link attempt when that provider account is already linked to a different user → 409
- **INT-010** — DELETE /auth/accounts/{account_id} → account row removed
- **INT-011** — DELETE /auth/accounts/{account_id} when it's the user's last account → 400 CANNOT_UNLINK_LAST_ACCOUNT
- **INT-012** — GET /auth/me → returns user profile + linked accounts list

## Key API Routes (from `apps/api/routers/auth.py`, prefix `/auth`)

- GET `/auth/login/{provider}` — redirects to OAuth provider
- GET `/auth/callback/{provider}` — OAuth callback, creates/finds user, returns JWT
- POST `/auth/logout` → 204
- GET `/auth/me` → user profile + accounts
- GET `/auth/link/{provider}` — link additional account (requires auth)
- GET `/auth/accounts` → list linked accounts
- DELETE `/auth/accounts/{account_id}` → unlink account
- GET `/health` (no prefix)

## Model/Token Info

- `User` model: `models.user.User` (id, display_name, email)
- `Account` model: `models.account.Account` (user_id, provider, provider_account_id, access_token)
- JWT creation: `core.jwt.create_access_token({"sub": str(user.id)})`

## Test Patterns

Each test must carry `@pytest.mark.integration` + the group marker (e.g. `@pytest.mark.int_health`).
Use the `AsyncClient` from `httpx` against the FastAPI `app` (use `httpx.AsyncClient(app=app, base_url="http://test")`).
Import `app` from `main`.

## OAuth Callback Flow Note

The `/auth/callback/{provider}` route expects a `code` query parameter and uses an httpx client internally to exchange the code for a token. The `mock_oauth` fixture (via `respx_mock`) intercepts those httpx calls. To trigger the callback, GET `/auth/callback/twitter?code=mock-code&state=...` — check the router to confirm what state param (if any) is required.
