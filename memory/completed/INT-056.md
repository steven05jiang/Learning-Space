## Requirements

BDD integration test for the core keyword search flow. Verifies that `GET /resources/search?q=<keyword>` returns a ranked list of the user's READY resources, excludes non-READY resources, and includes the `rank` field. Uses a real PostgreSQL test database with the GIN index in place.

### Acceptance Criteria

- [x] Test creates a set of READY resources with different titles/summaries/tags
- [x] Test creates at least one PENDING or FAILED resource (must be excluded)
- [x] `GET /resources/search?q=<keyword>` returns 200 + `{resources: [...], total: N}`
- [x] Returned resources are READY only
- [x] Results are ordered by relevance (rank DESC)
- [x] `rank` field present and > 0 on each result
- [x] Resources belonging to another user are not returned
- [x] Test marked `@pytest.mark.integration` and assigned to `search` CI group
