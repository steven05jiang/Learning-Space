## Requirements

BDD integration test for the core keyword search flow. Verifies that `GET /resources/search?q=<keyword>` returns a ranked list of the user's READY resources, excludes non-READY resources, and includes the `rank` field. Uses a real PostgreSQL test database with the GIN index in place.

## Implementation Summary

Created `apps/api/tests/integration/test_int_search.py` with 4 integration tests covering: READY-only filtering with PENDING/FAILED exclusion, rank DESC ordering with rank > 0, user isolation, tag filter, empty results, and unauthenticated 401. Also fixed 3 bugs in `resource_search_service.py` found during integration testing: (1) `tags ?? :tag` → `jsonb_exists(tags, :tag)` for asyncpg compatibility; (2) added `.bindparams(bindparam("tag", type_=String()))` to resolve AmbiguousParameterError when tag=None; (3) `str(owner_id)` → `owner_id` since owner_id is Integer not UUID.
