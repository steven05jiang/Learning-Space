## Requirements

BDD integration test verifying the `?tag=` filter parameter. Confirms that when a tag filter is provided, only resources containing that exact tag are returned (using `jsonb_exists(tags, :tag)`), even when multiple resources match the keyword query.

### Acceptance Criteria

- [ ] Test creates READY resources: some with target tag, some without
- [ ] `GET /resources/search?q=<keyword>&tag=<tag>` returns only resources containing that exact tag
- [ ] Resources matching keyword but missing the tag are excluded from results
- [ ] `?tag=nonexistent` returns empty list (total=0), not an error
- [ ] Test marked `@pytest.mark.integration` and assigned to `search` CI group
