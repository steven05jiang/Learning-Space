## Requirements

**Type:** Backend
**BDD Reference:** Scenario: User views resources for a graph node
**Description:** Query PostgreSQL for resources where `tags` contains the given tag name (node_id maps to a tag label) and `owner_id` matches the authenticated user. Return a list of resources.

**Acceptance Criteria:**
- [x] Returns resources matching the tag for the current user (owner_id scoped)
- [x] Uses GIN index on `tags` JSONB column for efficient query
- [x] Response includes `id`, `title`, `summary`, `original_content`, `content_type`, `status`
- [x] 404 if no matching resources (or empty list — acceptable)
- [x] 401 if unauthenticated

**Dependencies:** DEV-006 ✅, DEV-012 ✅

## Implementation Summary

New graph router at `apps/api/routers/graph.py` (104 lines). `ResourceNodeItem`/`ResourceNodeResponse` schemas added to `apps/api/schemas/resource.py`. Features: GIN index via PostgreSQL `@>` operator, regex validation on `node_id` (`^[\w\s-]+$`), configurable pagination (limit 1–100, offset ≥0), owner-scoped results ordered by `created_at` desc. 12 unit tests covering all scenarios including pagination boundaries and injection prevention. All CI checks pass.
