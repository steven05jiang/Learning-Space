## Requirements

Add a "Categories" section to the Settings page. Users can see all root categories (system and their own user-created ones), add custom categories, and delete their own (non-system) categories.

### Acceptance Criteria

- [x] Settings page has a "Categories" section (below or alongside the linked accounts section)
- [x] System categories listed with a lock icon; no delete button shown
- [x] User-created categories listed with a delete (×) button
- [x] Clicking delete shows a confirmation; on confirm calls DELETE /categories/{id}
- [x] "Add category" input field + submit button; calls POST /categories; shows validation error on duplicate name
- [x] Category list refreshes after successful add or delete (no full page reload)
- [x] Empty state shown when user has no custom categories

## Implementation Summary

Added Categories section to settings page with system (lock icon, no delete) and user categories (delete with confirmation). Implemented add category form with case-insensitive duplicate validation in both mock and API modes. Added 14 comprehensive tests covering all CRUD operations, error handling (409/403/404/401), and mock mode parity.
