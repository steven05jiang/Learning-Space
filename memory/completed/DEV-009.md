## Requirements

Implement `GET /auth/me` endpoint that:

- Uses `get_current_user` dependency (DEV-006) to authenticate the request
- Returns the current user's profile: id, email, display name, linked providers, created_at
- Returns HTTP 401 if not authenticated
- Response schema matches the frontend's expectation for logged-in state

**Depends on:** DEV-006 (auth middleware)

## Implementation Summary

GET /auth/me endpoint implemented using the `get_current_user` FastAPI dependency. Returns full user profile including id, email, display_name, avatar_url, created_at, and linked OAuth providers. Uses `selectinload(User.accounts)` to avoid N+1 queries. Returns 401 with WWW-Authenticate header if unauthenticated. 5 test cases covering success, missing auth, invalid token, multiple providers, and null timestamps.
