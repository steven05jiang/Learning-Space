## Requirements

Connect the existing graph visualization UI (already built with mock data in DEV-044/045) to the live backend APIs. The UI shell is complete — this task replaces mock data with real API calls.

### APIs to wire

| Endpoint | When | What to do with response |
|----------|------|--------------------------|
| `GET /api/graph` | On mount | Populate initial graph nodes + edges |
| `POST /api/graph/expand` | On node click (expand) | Add new child nodes + edges to graph |
| `GET /api/graph/nodes/{id}/resources` | On node click (panel) | Show resources in the resource panel dialog |

## Implementation Summary

Replaced hardcoded `knowledgeData` mock in `knowledge-graph.tsx` with real API calls. Added `useEffect` to fetch `GET /api/graph` on mount and populate graph state. On node click: calls `POST /api/graph/expand` and merges new nodes/edges into state; calls `GET /api/graph/nodes/{id}/resources` and displays in existing ResourcePanel dialog. Implemented loading spinner, empty state message ("No knowledge graph yet — add and process some resources"), and error toast handling. TypeScript types correctly mapped (API `label` → `name`, `level` → `category`, color by level). Net -107 lines (removed mock data). `npm run lint` and `npm run build` both pass.
