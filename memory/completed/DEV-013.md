## Requirements

Implement `POST /resources` endpoint that:

- Requires authentication via `get_current_user` dependency (DEV-006) ‚Äî return 401 if not authenticated
- Accepts request body with `content_type` (`url` or `text`), `original_content` (required), and `prefer_provider` (optional)
- Creates a resource row in PostgreSQL with: correct `owner_id` (from current_user), `content_type`, `original_content`, `prefer_provider`, `status=PENDING`
- Enqueues a background processing job for the resource
- Returns HTTP 202 with the full resource response (matching `ResourceResponse` Pydantic model from DEV-012)

**BDD Scenarios covered:**

- Authenticated user submits a URL resource ‚Üí 202, status PENDING/PROCESSING, job enqueued
- Authenticated user submits a text resource ‚Üí 202, status PENDING/PROCESSING
- Unauthenticated user attempts to create a resource ‚Üí 401
- User submits a URL with prefer_provider hint ‚Üí resource stored with prefer_provider for worker

**Acceptance Criteria:**

- [x] Resource row created with correct `owner_id`, `content_type`, `original_content`, `prefer_provider`
- [x] Status is PENDING or PROCESSING
- [x] Background job is enqueued
- [x] Returns 202 with full resource response
- [x] Returns 401 if not authenticated

**Depends on:** DEV-006 (merged PR #16), DEV-012 (merged PR #10), DEV-002 (merged PR #7)
