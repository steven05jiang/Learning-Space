## Requirements

Wire the existing `ChatPanel` component to the real `POST /chat` and `GET /chat/conversations` API endpoints, and remove the "coming soon" mode added by DEV-068.

### Context

- `apps/web/components/chat-panel.tsx` — the existing chat UI shell (read it carefully before changing)
- `apps/api/routers/chat.py` — new chat router (PR #222 + #224):
  - `POST /chat` — body: `{message: str, conversation_id?: UUID}` → response: `{conversation_id, message_id, response, role}`
  - `GET /chat/conversations` — returns `{items: [...], total, limit, offset}`
  - `GET /chat/conversations/{id}/messages` — returns `{id, messages: [...]}`
- Auth pattern: all other components use `localStorage.getItem("auth_token")` and pass `Authorization: Bearer <token>` header. Follow the same pattern.

### Changes needed in `chat-panel.tsx`

1. **Track `conversationId` state**: `const [conversationId, setConversationId] = useState<string | null>(null)`

2. **Remove "coming soon" initial message**: Replace the non-mock initial message (currently `"AI Chat is coming soon! Check back later."`) with an empty messages array `[]`.

3. **Fix the POST /chat API call**:
   - Include `conversation_id: conversationId` in the request body (only if not null)
   - On success, extract `data.conversation_id` and call `setConversationId(data.conversation_id)` to persist the conversation across turns
   - Extract the assistant response from `data.response`

4. **Enable input in non-mock mode**: Remove `!isMock` from both disabled conditions:
   - Input: `disabled={isLoading}` (was `disabled={isLoading || !isMock}`)
   - Button: `disabled={!input.trim() || isLoading}` (was `disabled={!input.trim() || isLoading || !isMock}`)

5. **Update error message**: Replace "coming soon" fallback error text with a neutral error message like `"Sorry, I couldn't reach the AI service. Please try again."`

6. **Do NOT remove mock mode support** — keep the `useMock()` branch intact for local development

### What NOT to change

- Do not change the UI layout, styling, or animation
- Do not add conversation history loading on panel open (that's out of scope)
- Do not change the `/agent/query` endpoint — the new chat panel should use `POST /chat`
- Do not modify any other files outside of `chat-panel.tsx`

### Acceptance criteria

- Input and Send button are enabled in non-mock (production) mode
- Sending a message calls `POST /chat` with `{message, conversation_id?}`
- First message creates a new conversation; subsequent messages send the `conversation_id` to continue it
- Assistant response displayed correctly from `data.response`
- Error case shows neutral error message (no "coming soon" text)
- Mock mode still works as before
- `make web-lint` passes
