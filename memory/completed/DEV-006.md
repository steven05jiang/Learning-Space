## Requirements

Implement a FastAPI dependency `get_current_user` that:
- Reads the session cookie / JWT set by the OAuth login flow (DEV-005)
- Validates the token and resolves the current user from the database
- Returns the authenticated user object for use in protected endpoints
- Raises HTTP 401 if the token is missing, invalid, or expired
- Is used as a dependency injected into all protected routes

**Depends on:** DEV-005 (OAuth login flow — complete, PR #13)

## Implementation Summary

FastAPI `get_current_user` dependency using JWT validation via HTTPBearer. Validates token, checks user existence in database, raises HTTP 401 with WWW-Authenticate headers for all failure modes. 6 comprehensive test cases. Security verified — no injection vulnerabilities, proper auth patterns.
