## Requirements

Implement `POST /graph/expand` — allows the frontend to drill down into a specific graph node and get its neighbors (child nodes + edges).

### Request body

```json
{ "node_id": "AI", "direction": "out" }
```

- `node_id` (required): the tag name to expand
- `direction` (optional): `"out"` (default), `"in"`, `"both"`

### Response shape (must match frontend mock contract)

```json
{
  "nodes": [
    { "id": "Python", "label": "Python", "level": "child" }
  ],
  "edges": [
    { "source": "AI", "target": "Python", "weight": 3 }
  ]
}
```

## Implementation Summary

Added `GraphExpandRequest` schema with `@field_validator` for direction validation (rejects invalid values with 422). Added `get_neighbors(owner_id, node_id, direction)` to `GraphService` with distinct Neo4j queries for `"out"` (->), `"in"` (<-), and `"both"` (-) directions. Added `POST /graph/expand` route reusing existing graph schemas. 6 unit tests covering all direction values, invalid direction, unknown node, and authentication. Required 2 review rounds — round 1 fixed unused direction parameter and missing direction validation; approved on round 2.
