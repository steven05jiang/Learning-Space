# DEV-071: Multi-LLM provider abstraction

**Status:** ✅ Completed
**Priority:** High
**Completed:** 2026-03-27
**PR:** #164 (merged)

## Requirements

Abstract the LLM client behind a provider interface so a single `LLM_PROVIDER` env var
selects between Anthropic, Groq, SiliconFlow, and Fireworks AI.

## Implementation Summary

- Added `llm_provider` env var + per-provider API keys and model overrides to `core/config.py`
- Created `services/llm_client.py` with `get_llm_client()` factory (LangChain-compatible)
- Added `get_direct_client()` for non-LangChain usage
- Updated agent service to use factory instead of direct `ChatAnthropic`
- Added `langchain-groq` and `langchain-openai` dependencies
- Comprehensive test coverage; all 391 tests pass

**Providers supported:**
- Anthropic: `claude-haiku-4-5-20251001` (default)
- Groq: `llama-3.1-8b-instant`
- SiliconFlow: `Qwen/Qwen2.5-7B-Instruct`
- Fireworks: `accounts/fireworks/models/llama-v3p1-8b-instruct`

**Cost impact:** Groq/SiliconFlow/Fireworks ~$0.07–0.10/MTok vs Anthropic ~$0.25/MTok
