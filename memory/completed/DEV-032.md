## Requirements

Implement a LangGraph-based conversational agent that can answer questions about the user's saved resources.

- Agent graph: user message → tool selection → tool execution → response
- Tools the agent must have:
  - `search_resources(query)` — full-text or tag-based search over user's resources (via GET /resources with filters)
  - `get_graph_context(tag)` — retrieve related tags/nodes from user's knowledge graph
  - `get_resource_detail(resource_id)` — fetch full resource content/summary
- Agent must be multi-turn: accepts conversation history as context
- LLM: use the configured model from settings (ANTHROPIC_MODEL env var, default claude-haiku-4-5-20251001)
- Use LangGraph StateGraph with a checkpointer (in-memory for now — persistence via DEV-035 conversation storage to be wired in DEV-033)
- Unit tests: mock LLM + tool calls; verify agent routes to correct tools and returns coherent response
- Follow existing patterns in `apps/api/` for services (see services/ directory)

Depends on: DEV-014 ✅, DEV-025 ✅, DEV-004 ✅, DEV-035 (dispatch after DEV-035 merges)
