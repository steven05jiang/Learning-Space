## Requirements

**Type:** Backend
**BDD Reference:** Scenario: User links an additional social account; Scenario: User attempts to link an account already linked to another user
**Description:** Implement `GET /auth/link/{provider}` (requires auth). Encodes `link:<user_id>` in OAuth state. On callback, if state indicates link flow, create new `user_accounts` row for current user. If `(provider, external_id)` exists for another user, return 409.

**Acceptance Criteria:**

- [x] Authenticated user can start link flow for any supported provider
- [x] New `user_accounts` row is created under the current user
- [x] 409 returned if account already belongs to another user
- [x] Tokens are stored for the linked account

**Estimated Effort:** M (~40min)
**Dependencies:** DEV-005 ✅, DEV-006 ✅

## Implementation Summary

Implemented OAuth account linking flow with `GET /auth/link/{provider}` endpoint requiring authentication and encoding `link:<user_id>` in OAuth state. The callback handler was updated to detect link vs login flows via state prefix. Fixed a critical security vulnerability found in review: the callback endpoint was not enforcing authentication for link flows, allowing cross-user account hijacking. Fixed by adding an optional `get_current_user` dependency and verifying `current_user.id == link_user_id` before proceeding. Added comprehensive test coverage (104 total tests passing).
