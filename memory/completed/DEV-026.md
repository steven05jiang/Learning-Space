## Requirements

When a resource is processed by the worker, the knowledge graph must be updated with the resource's tags (create Tag nodes + RELATED_TO edges in Neo4j). This closes the loop between resource processing (DEV-023) and the graph service (DEV-025).

**Note:** `tasks.py` `process_resource()` already calls `graph_service.update_from_resource(resource.owner_id, llm_result.tags)` at Step 5 (labeled "placeholder for DEV-026"). The implementer should:
1. Verify this hook is correct — check that `owner_id` type matches (int vs UUID), tags are passed correctly, and error handling is appropriate (currently wraps in try/except so graph failure doesn't fail the job — this is correct behavior).
2. Confirm the `sync_graph` stub function (used for deletion) needs upgrading for DEV-027 — but that is a separate task; do NOT implement DEV-027 here.
3. Write/update unit test in `tests/test_process_resource.py` to assert that `graph_service.update_from_resource` is called with the correct arguments after successful LLM processing.
4. Run `make api-lint` and `make api-test` — both must pass.
